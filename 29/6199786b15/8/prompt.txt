# Fix: ProcessingIndicator not visible on recipe preview

## Context

When a recipe is pending or processing, the `ProcessingIndicator` component is correctly mounted (the same `recipe.status` check that disables the edit button also gates the indicator), but it renders as 0x0 pixels due to a layout collapse.

## Root Cause

In `RecipeViewScreen.tsx` (line 95), the wrapper `<View pointerEvents="none">` has **no style**. All its children use `position: "absolute"`:

- `MetadataOverlay` â€” absolute, but specifies its own height via padding/content, so it overflows and remains visible
- `ProcessingIndicator` â€” absolute with `top:0, left:0, right:0, bottom:0`, so its dimensions are **derived from the parent**. Parent is 0x0 â†’ indicator is 0x0
- `DebugVisualization` â€” absolute

Since no child contributes intrinsic height, the parent View collapses to 0x0. ProcessingIndicator inherits that zero size.

## Fix

Add `StyleSheet.absoluteFillObject` to the wrapper View at line 95 of `features/recipe-preview/components/RecipeViewScreen.tsx`:

```tsx
// Before:
<View pointerEvents="none">

// After:
<View pointerEvents="none" style={StyleSheet.absoluteFillObject}>
```

This makes the View fill the overlay Animated.View. Since it's absolutely positioned, it doesn't affect the sibling button layout. `pointerEvents="none"` already ensures touches pass through to the buttons.
