# Plan: Task Detail Expansion Script

## Context
The Simple Task Master has 8 task files in `.simple-task-master/tasks/` with brief descriptions. We need a script that spawns a Claude Code session for each task to analyze the codebase and write a detailed implementation plan back into the task file.

## Approach
Create a bash script `scripts/expand-tasks.sh` that:

1. Iterates over all `.md` files in `.simple-task-master/tasks/`
2. For each task file, spawns a Claude Code session using `claude -p` (print mode) with `--dangerously-skip-permissions`
3. The prompt instructs Claude to:
   - Read the task file to understand the issue
   - Explore the codebase to find relevant files and patterns
   - Think about what the issue means specifically in this codebase
   - Plan the solution
   - Output **ONLY** the implementation plan as markdown (no frontmatter, no original body)
4. The script **preserves the original file entirely** and appends the plan output under a `## Implementation Plan` section
5. Runs tasks in parallel (background processes) with a concurrency limit

## File Structure Preservation

Task files have this structure:
```
---
frontmatter (YAML)
---

Original body text (user's notes)
```

The script will:
- Read the original file completely
- Strip any existing `## Implementation Plan` section (for re-runs/idempotency)
- Append `\n\n## Implementation Plan\n\n` + Claude's output
- This way frontmatter, status, tags, dependencies, and original description are **never touched**

## Key Details

- **Claude flags**: `-p` (non-interactive print mode), `--dangerously-skip-permissions` (no approval prompts for read-only exploration)
- **Allowed tools**: `--tools "Read,Grep,Glob,Bash(ls:*),Bash(find:*),Agent"` — read-only tools only, no Edit/Write
- **Model**: Default model, overridable via `CLAUDE_MODEL` env var
- **Output handling**: Claude outputs ONLY the plan text; script appends it to the preserved original file
- **Concurrency**: Run up to 3 tasks in parallel to balance speed vs API limits
- **Idempotency**: If re-run, strips previous `## Implementation Plan` before appending new one
- **Safety**: The script only modifies task files in `.simple-task-master/tasks/`, never codebase files
