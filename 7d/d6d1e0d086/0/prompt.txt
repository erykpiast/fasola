# Improve title extraction: burst detection + full scan + multi-title

## Context

The current `extractTitleWithEmbeddings` scans only the first ~10 non-empty lines. Real cookbook OCR pages often start with a burst of short garbled lines from a partially visible adjacent page, pushing real titles out of the window. Pages can also contain multiple recipes.

## Approach

1. Detect and skip the initial burst of short/garbled lines
2. Scan all remaining lines for candidates (with smart cap to limit embed calls)
3. Return multiple titles joined with " + "

## File: `lib/text-classifier/title-extractor.ts`

### Step 1: Add burst detection

A "burst" = consecutive short lines (< 20 chars) at the start of the text. Skip them to find where real content begins.

```
function findBurstEnd(lines: Array<{text: string}>): number
```

Walk lines from the start. While a line is short (< 20 chars), keep going. Stop when we hit a line >= 20 chars. Return that index as the starting point for candidate generation.

Edge case: if the very first line is long, burst length is 0 (no burst).

### Step 2: Replace `buildCandidates` — full scan after burst

After skipping the burst, scan ALL remaining non-empty lines. For each line that passes hard filters:
- Single-line candidate
- 2-line join (if joined length ≤ 80)
- 3-line join (if joined length ≤ 80)

Track `position` (original line index) for sorting at the end.

### Step 3: Prioritize candidates, cap at ~25

If more candidates than 25, prioritize:
1. ALL_CAPS lines (strong cookbook title signal)
2. Short lines (≤ 5 words)

Deduplicate by normalized text, slice to 25.

### Step 4: Add noise reference embedding

Add `NOISE_REFERENCE = "page number, table of contents, book footer, garbled text"` as a third cached reference.

Score: `titleSim - max(headerSim, noiseSim)` — no position bonus (unreliable with full scan).

### Step 5: Multi-title selection

Instead of returning single best:
1. Filter to `score >= max(0.05, bestScore * 0.6)` — absolute floor + relative threshold
2. Deduplicate: if one title is substring of another, keep the longer one
3. Cap at 3 titles
4. Sort by document position (reading order)
5. Join with `" + "`

### Step 6: Update `_resetEmbeddingCacheForTests`

Reset `cachedNoiseRefEmbedding` too.
