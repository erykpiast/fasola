# Fix: Tag disappears after being added to search input

## Context

Commit `1ad38be` ("widen search input to prevent placeholder text truncation") changed the text field's `idealWidth` from `120` to `geometry.size.width` to prevent Polish placeholder text from being truncated. This introduced a side effect: when a tag is added, the text field still occupies the full container width, making the total HStack content (tag pills + full-width text field) wider than the visible scroll area. The `onChange(of: selectedTagIdSignature)` handler then auto-scrolls to `.trailing` of "input-tail", pushing the newly added tag pill completely out of view.

## Fix

**File:** `modules/liquid-glass/ios/LiquidGlassInputView.swift` (line 414)

Change the `idealWidth` to be conditional — full width only when no tags are present (to prevent placeholder truncation), smaller when tags exist (so tags + input fit without overflowing):

```swift
// Before:
.frame(minWidth: 80, idealWidth: geometry.size.width)

// After:
.frame(minWidth: 80, idealWidth: selectedTags.isEmpty ? geometry.size.width : 120)
```

This restores the original `idealWidth: 120` behavior when tags are present, while keeping the Polish placeholder fix for the empty/no-tags state.

4. Remove all tags → placeholder text should display without truncation (especially in Polish)
5. Type text with no tags → text field should use full width, no truncation
