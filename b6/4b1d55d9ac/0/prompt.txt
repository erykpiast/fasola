# Fix: Make processing transparent â€” show edit/remove buttons immediately

## Context

When a user imports a photo and opens it before background processing completes, `RecipeViewScreen` shows a redundant source selector + confirm button (the source was already set during import). The user is stuck with no back button and a confusing repeated workflow. Processing should be transparent: always show the normal recipe view (back, delete, edit) with edit disabled during processing, and display fun rotating status messages over the photo.

## Changes

### 1. `features/recipe-preview/components/RecipeViewScreen.tsx`

Remove the split `isProcessing` / `isReady` branches. Unify into a single view that always shows:
- `RecipeImageDisplay`
- `MetadataOverlay` (will show source, and title/tags once processing completes)
- `BackButton`
- `DeleteButton`
- `EditButton` with `disabled={!isReady}`
- When `isProcessing`: a centered `ProcessingIndicator` component (spinner + rotating message)

Remove: `SourceSelector`, `ConfirmButton`, `ConfirmButtonRef` imports and all related logic (`handleSourceChange`, `handleConfirm`, `confirmButtonRef`). These are no longer needed in this screen.

### 2. `lib/components/atoms/EditButton.tsx`

Add optional `disabled` prop. When true: set `opacity: 0.4` and `pointerEvents: 'none'` on the container style. No native module changes needed.

```tsx
export function EditButton({ onPress, disabled }: { onPress: () => void; disabled?: boolean }): JSX.Element {
  return (
    <LiquidGlassButton
      onPress={onPress}
      systemImage="pencil"
      style={[styles.button, disabled && styles.disabled]}
      accessibilityLabel="Edit"
    />
  );
}
// styles.disabled = { opacity: 0.4, pointerEvents: 'none' }
```

### 3. New: `features/recipe-preview/components/ProcessingIndicator.tsx`

Component showing `ActivityIndicator` + rotating `Text` message centered on screen. Uses a `useEffect` interval (2s) to cycle through messages.

Logic:
- Define a `FIRST_MESSAGE` key (always shown first)
- Define array of ~10 `ROTATING_MESSAGES` keys
- On mount, shuffle the rotating messages array (Fisher-Yates)
- Start with `FIRST_MESSAGE`, then cycle through shuffled array every 2s
- After exhausting shuffled array, loop back to start of shuffled array (skip first message on subsequent loops so the user discovers new ones)
- Fade transition between messages using `Animated.Text` opacity
