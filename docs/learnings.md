# Development Learnings

Distilled knowledge from development sessions -- problems encountered, solutions that worked, and patterns to follow or avoid. This file is maintained by Claude Code and committed to the repo so insights persist across sessions and contributors.

## How This File Is Maintained

After resolving a non-trivial issue or receiving a correction, Claude Code appends a short entry here capturing:
- What went wrong or was surprising
- What fix or approach worked (or didn't)
- Any general takeaway
- UX/design preferences the user corrected
- Code patterns the user preferred over what was initially produced

Entries are grouped by topic. Outdated or incorrect entries should be updated or removed. When a preference solidifies into a hard rule, it should be promoted to `AGENTS.md`.
